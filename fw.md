**校园服务智能体项目范围文档**

**1. 项目名称**

**面向学生的校园服务智能体平台**

**2. 项目背景**

当前高校学生在获取校园信息时，通常面临以下问题：

1.  学校规章制度、办事流程、通知公告分散在多个系统或网页中，查找成本高。

2.  学生对教师信息、课程安排、选课建议、课程规划等内容缺乏统一入口。

3.  校园帖子、问答、经验分享内容质量参差不齐，信息可信度不足。

4.  校园地图、办事地点、功能部门位置等导航需求频繁，但查询体验较差。

5.  传统搜索方式只能"找文档"，不能结合上下文给出面向学生场景的解释与建议。

基于大语言模型与检索增强生成技术，可以构建一个集
**校园知识问答、事务辅助、课程规划、选课建议、帖子辅助与校园导航**
于一体的校园服务智能体，提升学生获取信息与完成事务的效率。

**3. 项目目标**

建设一个面向学生的一站式校园服务智能体，具备以下目标：

- 为学生提供准确、可追溯的校园信息问答能力。

- 基于学校规章制度、通知公告、教师信息等校内知识，减少错误回答与幻觉。

- 融合课表、课程规划、选课建议、校园导航等高频场景，提升服务实用性。

- 提供统一对话式入口，降低学生查找与理解校园信息的门槛。

- 为后续扩展到教师端、辅导员端、行政端打下基础。

**4. 项目定位**

本项目定位为：

**以 RAG
为核心知识引擎、以学生事务场景为主线、以多功能校园服务为扩展方向的智能体平台。**

它不是单一的聊天机器人，而是一个具备：

- 校园知识检索问答能力

- 学生事务辅助能力

- 个性化推荐与规划能力

- 场景化工具调用能力

的综合型校园服务系统。

**5. 目标用户**

**5.1 核心用户**

- 在校本科生

- 在校研究生

- 新生

**5.2 次级用户**

- 辅导员

- 教务老师

- 学校行政服务人员

- 校园信息平台主管理员

**6. 业务范围**

**6.1 项目范围内功能**

**模块一：校园知识问答**

基于学校知识库提供准确问答，覆盖以下内容：

- 学校规章制度

- 学校通知公告

- 办事流程

- 学生手册

- 奖助学金政策

- 宿舍管理规定

- 考试管理规定

- 请假、休学、复学、转专业等流程

- 教师基本信息

核心能力：

- 支持自然语言提问

- 支持多轮对话

- 支持答案出处引用

- 支持相近问题推荐

- 支持知识更新后快速生效

**模块二：校园帖子辅助**

面向学生社区或校园论坛内容提供辅助能力：

- 帖子内容摘要

- 相似问题归类

- 敏感或违规内容初筛

- 根据校规/通知对帖子内容进行事实辅助解释

- 热点问题聚类

核心目标：

- 提升帖子阅读和管理效率

- 将"学生经验"与"官方知识"区分展示

- 避免未经验证的帖子误导学生

**模块三：课表服务**

围绕学生个人课表提供服务：

- 课表查看

- 今日课程提醒

- 教室位置查询

- 课程时间冲突识别

- 考试时间关联展示

- 上课地点导航联动

**模块四：课程规划**

提供中长期学习路径辅助：

- 按培养方案展示课程结构

- 必修/选修课程分类

- 学分完成度统计

- 毕业要求进度提示

- 学期课程建议分配

- 个性化修课路径建议

**模块五：选课建议**

结合学生背景与规则进行辅助推荐：

- 基于培养方案判断是否符合选课前置条件

- 结合教师信息、课程评价、课程难度提供建议

- 提供冲突提醒

- 提供不同选课目标下的推荐方案\
  例如：

  - 保稳型

  - 提升型

  - 兴趣型

  - 考研导向型

  - 就业导向型

注意：\
该模块输出的是"建议"，不替代学校正式教务系统决策。

**模块六：校园导航**

提供校园空间与服务点导航：

- 教学楼、图书馆、宿舍、食堂、行政楼等地点查询

- 办公室、窗口、服务中心位置导航

- 上课地点路径规划

- 结合校园通知提供场景导航\
  例如：新生报到流程对应地点导航

**模块七：统一智能体交互层**

作为各能力统一入口：

- 对话式问答

- 任务型引导

- 工具调用

- 个性化上下文记忆

- 多意图识别\
  例如：

  - "奖学金申请截止时间是什么时候？"

  - "王老师是谁，教什么课？"

  - "我下学期怎么选课比较合适？"

  - "从宿舍去明德楼怎么走？"

**6.2 不在当前范围内功能**

以下内容建议暂不纳入第一期：

- 自动代替学生完成正式选课操作

- 自动修改教务系统数据

- 自动审批行政流程

- 医疗、心理咨询等高风险专业建议

- 涉及隐私的学生成绩深度预测

- 支持校外开放用户的完整访问

- 多校区、多学校通用化适配

这些可作为后续扩展方向，但不建议在首版范围中同时推进。

**模块八：校园申请文书智能辅助**

**1. 功能描述**

将学校活动申请、学生请假、奖学金申请等相关规范文件纳入知识库，供智能体检索调用，帮助学生自动生成符合格式规范的申请材料初稿，并进行合规检查和优化建议。

**2. 适用文书类型**

包括但不限于：

- 活动申报书

- 请假条

- 奖学金申请表

- 困难补助申请材料

- 社团活动申请材料

- 教室借用申请

- 学生证明申请说明

- 其他校园常见申请文本

**3. 核心能力**

- 基于学校规范文件进行检索增强生成

- 根据申请类型自动匹配对应模板

- 引导学生补充必要信息

- 自动生成符合规范的申请文书初稿

- 检查格式是否完整

- 检查是否缺少关键字段

- 优化语言表达，使其正式、清晰、规范

- 给出"依据的学校规范来源"或"参考模板来源"

**4. 典型使用场景**

- 学生需要写活动申报书，但不清楚格式和审批要求

- 学生想写请假条，但不知道请假原因、时间、课程影响说明该如何表达

- 学生填写奖学金申请表时，不知道个人陈述如何撰写

- 学生已经写好申请材料，希望系统帮助检查是否符合学校要求

**5. 输出形式**

系统可输出以下结果：

- 规范化文书初稿

- 表单填写建议

- 缺漏项提醒

- 格式检查结果

- 修改建议

- 对应规范依据说明

**6. 边界说明**

该模块用于
**辅助生成和规范检查**，不直接代替学校审批，不保证申请一定通过。最终提交内容仍需学生本人确认，并以学校正式要求为准。

**7. 数据范围**

**7.1 核心数据源**

**结构化数据**

- 教师信息表

- 课程信息表

- 课表数据

- 培养方案数据

- 教学楼/校园地点数据

- 通知元数据

- 学校部门信息

**非结构化数据**

- 学校规章制度文档

- 学校通知公告文档

- 学生手册

- 办事指南

- 常见问题文档

- 校园帖子文本内容

**7.2 数据分层**

将数据分为三层：

**第一层：权威官方知识**

- 学校规章制度

- 教务处/学工处/学院公告

- 官方办事流程

- 培养方案

- 正式教师信息

用途：

- RAG 核心知识库

- 高可信回答来源

**第二层：业务系统数据**

- 课表

- 课程信息

- 教室与地点

- 学分与课程结构

用途：

- 工具调用

- 个性化服务

- 课程规划与课表展示

**第三层：社区与辅助数据**

- 校园帖子

- 学生经验分享

- 课程评价

- 选课经验

用途：

- 作为参考信息补充

- 需与官方信息分开展示

**8. 技术范围**

**8.1 核心技术路线**

本项目采用：

**大语言模型 + 检索增强生成（RAG） + 工具调用 + 结构化服务接口**

**技术组成建议**

- 前端：Web 或小程序/APP

- 后端：API 服务

- 知识库：向量数据库 + 文档索引

- 模型层：大语言模型

- 检索层：混合检索/向量检索/重排

- 工具层：课表查询、教师查询、导航查询、课程规划接口

- 数据层：关系数据库 + 文件存储 + 向量数据库

**8.2 RAG 使用范围**

RAG 主要用于以下场景：

- 校规制度问答

- 学校通知解释

- 办事流程说明

- 教师信息问答

- 官方知识与帖子内容对照解释

RAG 的目标是：

- 保证回答可溯源

- 降低模型幻觉

- 支持知识快速更新

**8.3 智能体能力范围**

智能体能力主要包括：

- 意图识别

- 知识问答

- 工具调用

- 多轮对话管理

- 上下文理解

- 个性化推荐输出

**9. 功能边界说明**

**9.1 回答边界**

系统应优先基于学校官方知识回答；当没有权威依据时，应明确说明：

- 当前无官方依据

- 以下内容仅供参考

- 建议联系相关部门确认

**9.2 推荐边界**

在课程规划和选课建议中：

- 可提供分析和建议

- 不直接替代学生做决策

- 不承诺课程一定能选上

- 不保证推荐方案一定最优

**9.3 导航边界**

在校园导航中：

- 可提供地点位置与推荐路线

- 不保证实时路况

- 室内精细导航可后续扩展

**10. 第一阶段 MVP 范围**

建议第一阶段只做最核心、最有价值的能力，控制范围。

**MVP 建议包含**

1.  校园知识问答

2.  学校规章制度/通知公告知识库

3.  教师信息查询

4.  基础课表查询

5.  简单课程规划建议

6.  基础校园导航查询

7.  问答引用来源展示

**MVP 暂不包含**

- 自动选课

- 帖子内容深度运营

- 高度个性化推荐系统

- 多模态地图导航

- 与多个校内系统深度打通

**11. 第二阶段扩展范围**

- 校园帖子摘要与分类

- 课程评价聚合

- 更细粒度课程规划

- 个性化选课画像

- 校园地图可视化导航

- 常办事项流程自动引导

- 新生入学助手

- 考试周专项服务

- 多终端接入

**12. 关键非功能需求**

**12.1 准确性**

- 基于权威知识源回答

- 关键回答需带出处

- 支持人工审核与知识更新

**12.2 可用性**

- 响应时间可控

- 常见问题秒级回答

- 页面简洁，适合学生使用

**12.3 可扩展性**

- 支持新增知识库

- 支持新增工具模块

- 支持按学院或校区扩展

**12.4 安全性**

- 用户身份鉴权

- 数据访问权限控制

- 敏感数据脱敏

- 日志审计

**12.5 隐私合规**

- 个人课表与选课偏好属于隐私数据

- 仅在用户授权前提下使用个性化信息

- 校园帖子处理需符合平台管理要求

**13. 风险与限制**

**13.1 数据风险**

- 校规与通知更新频繁，若知识库不同步可能回答过期

- 教师信息与课表数据存在时效性问题

**13.2 模型风险**

- 大模型可能产生幻觉

- 对复杂制度条款解释可能不完全准确

**13.3 系统接入风险**

- 教务系统、门户系统、地图系统接口可能不开放

- 数据权限申请可能耗时较长

**13.4 运营风险**

- 学生对"建议"误认为"官方结论"

- 帖子辅助功能可能面临内容审核压力

**14. 成功指标**

可从以下维度衡量项目效果：

**使用指标**

- 日活跃用户数

- 周活跃用户数

- 人均提问次数

- 功能模块使用占比

**效果指标**

- 问答命中率

- 引用覆盖率

- 用户满意度

- 错误回答率

- 人工转接率

**业务指标**

- 学生查找制度与通知的时间下降

- 课表与选课相关咨询量下降

- 校园办事效率提升

- 新生常见问题重复咨询减少
