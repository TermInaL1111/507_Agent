## 16）接口详细规格（前端联调版，完整）

> 参考来源：
> - `front/src/config/api.js`
> - `backend/api.md`
> - `front/api.md`（DjangoUserService）
> - `fw.md`
>
> 说明：
> - 本节按“前端可直接联调”的粒度提供：`api.js键名 / URL / 方法 / 鉴权 / 请求参数 / 响应结构 / 页面使用点 / 状态`。
> - **已实现**接口给出当前可用结构；**待新增**接口给出建议契约（用于前后端并行开发）。

### 16.1 通用约定

1. **基础地址**
   - 业务后端：`apiConfig.baseURL`
   - 用户服务：`apiConfig.userBaseURL`

2. **鉴权头**（除登录/注册等公开接口外）
   - `Authorization: Bearer <JWT>`

3. **返回包装约定**
   - FastAPI接口通常：
     ```json
     { "code": 200, "message": "success", "data": {...} }
     ```
   - Django用户接口常见：
     - 登录/注册：`{ message, user, token }`
     - 详情：`{ success, message, data }` 或 `{ message, user, token }`

4. **错误约定（建议前端统一处理）**
   - 401：登录失效/Token过期
   - 403：越权访问
   - 400：参数校验失败
   - 500：服务内部错误
